# Вычисление изолиний триангулированной поверхности

Программа вычисляет изолинии (линии уровня) на триангулированной поверхности методом marching triangles и строит замкнутые полилайны, представляющие контуры сечения.

## Описание задачи

**Изолиния** — линия на поверхности, вдоль которой значение функции постоянно. В данном проекте решается задача:

1. Дана триангулированная поверхность: множество точек с координатами (x, y, z) и триангуляция, соединяющая эти точки в треугольники
2. Задан уровень сечения `z_plane`
3. Требуется найти все замкнутые полилайны, образующие пересечение поверхности с плоскостью z = z_plane

Программа работает с произвольными триангуляциями и корректно обрабатывает вырожденные случаи: вершины на уровне сечения, плато, хребты, изолированные пики.

## Алгоритм

Алгоритм состоит из шести последовательных этапов:

1. **Бинаризация вершин** — классификация всех вершин относительно уровня z_plane (выше/ниже). Это упрощает проверку пересечения треугольников с плоскостью.

2. **Генерация сегментов** — для каждого треугольника, пересекающего плоскость, вычисляются две точки пересечения и создаётся направленный сегмент между ними. Направление определяется ориентацией треугольника, что гарантирует согласованность всех сегментов.

3. **Фильтрация вырождений** — удаляются петли (сегменты с совпадающими концами, возникающие когда вершина находится точно на z_plane) и аннигилируются взаимообратные пары сегментов. На выходе получается граф направленных рёбер, где для каждого узла входящая степень равна исходящей.

4. **Построение графа** — сегменты преобразуются в направленный граф смежности. Проверяется корректность: in_degree == out_degree для всех узлов, что является необходимым условием для существования Эйлеровых циклов.

5. **Разрешение седловых точек** — узлы с несколькими входящими/исходящими рёбрами (развилки) разрешаются алгоритмом "первое по часовой стрелке". Для каждого входящего ребра выбирается соответствующее исходящее на основе углов между направлениями, что даёт естественное продолжение изолинии без резких поворотов.

6. **Сборка полилайнов** — выполняется обход графа с построением замкнутых последовательностей точек. Каждый обход даёт одну замкнутую изолинию.

Результат работы программы: набор замкнутых полилайнов, представляющих изолинии поверхности.

## Сборка и запуск

**Требования:**
- Windows 10/11
- Visual Studio 2019+ (MSVC компилятор)
- Для полного генератора: собранная библиотека CDT

**Среда:** Visual Studio Developer PowerShell

### Основное приложение (вычисление изолиний)

```powershell
# Компиляция
cl /EHsc /O2 /std:c++17 /nologo intersectionCalculator.cpp /Fe:intersectionCalculator.exe

# Запуск
.\intersectionCalculator.exe test_data.bin
```

Создаёт файлы `segments.bin` (промежуточные сегменты) и `isolines.bin` (финальные изолинии).

### Полный генератор тестовых данных (50k вершин)

**Требует:** CDT библиотека собрана в `external/CDT/CDT/build/Release/CDT.lib`

```powershell
# Компиляция (x64)
cmd /c '"C:\Program Files\Microsoft Visual Studio\2022\...\vcvarsall.bat" x64 >nul 2>&1 && cl /I.\external\CDT\CDT\include /I.\external\CDT\CDT\build /EHsc /O2 /MD /std:c++17 /nologo generate_test_data_cdt.cpp /link external\CDT\CDT\build\Release\CDT.lib /OUT:generate_test_data_cdt.exe'

# Запуск
.\generate_test_data_cdt.exe
```

Создаёт `test_data.bin` (~2.5 МБ) и `test_data_report.txt`.

### Быстрый генератор (9 вершин)

```powershell

.\generate_simple_test.exe
```

## Формат данных

### test_data.bin

Бинарный файл с триангулированной поверхностью:

```
int32   N                    // Количество вершин
int32   M                    // Количество треугольников
double  vertices[N*2]        // Координаты (x, y)
double  zValues[N]           // Высоты z
int32   triangles[M*3]       // Индексы вершин (CCW)
double  z_plane              // Уровень сечения
uint8   boundary_mask[N]     // Граничные вершины
uint8   plateau_mask[N]      // Вершины плато
uint8   ridge_mask[N]        // Вершины хребта
int32   isolated_peak_idx    // Индекс изолированного пика (-1 если нет)
```

### segments.bin

Промежуточный результат — направленные сегменты:

```
int32   num_segments         // Количество сегментов
int32   num_points           // Количество уникальных точек
// Для каждой точки:
  int64   id                 // ID точки (>= 0: вершина, < 0: точка на ребре)
  double  x, y               // Координаты

// Для каждого сегмента:
  int64   start_id           // ID начальной точки
  int64   end_id             // ID конечной точки
```

**Кодирование ID:**
- Положительные ID: вершины триангуляции (совпадают с индексом)
- Отрицательные ID: точки на рёбрах, `encode_edge(u, v) = -(u*1000000 + v + 1)`

### isolines.bin

Финальный результат — замкнутые изолинии:

```
int32   num_polylines        // Количество изолиний
// Для каждой изолинии:
  int32   num_points         // Количество точек в полилайне
  double  points[num_points*2]  // Координаты (x, y)
```

## Тестовый пример

Полный генератор создаёт сложную триангулированную поверхность:

- **Область:** квадрат [-100, 100]² с вырезанной пятиконечной звездой
- **Триангуляция:** Constrained Delaunay с ~50,500 вершинами и ~100,000 треугольников
- **Рельеф:** синусоидальная функция `z = 5 * sin(0.05 * sqrt(x² + y²))`
- **Особые зоны:**
  - Плато: кольцо радиусов [35, 45], z = 0
  - Хребет: полоса x ∈ [-80, -60], y ≈ 80, z = 0
  - Изолированный пик: случайная точка, z = 0
  - Граница: внешняя и внутренняя (звезда), z = 1000
- **Уровень сечения:** z_plane = 0.0
- **Seed:** 42 (для воспроизводимости)

Особые зоны создают вырожденные случаи для проверки корректности алгоритма.

## Технические детали

- **C++ Standard:** C++17
- **Компилятор:** MSVC (cl.exe), флаги `/std:c++17 /O2 /EHsc`
- **Зависимости:** только STL в основном приложении; CDT только в генераторе
- **Ориентация:** треугольники ориентированы против часовой стрелки (CCW)
- **Точность:** `1e-10` для сравнения высот с z_plane
- **Binary I/O:** прямая запись/чтение структур, little-endian
